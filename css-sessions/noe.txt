- Cascading Style Sheet

- CSS-3

- It is used to provide look and feel to a webpage 

- There Three Different ways we can apply style to a webpage

  1. Inline Style

  2. Internal Style

  3. External style 


- Inline Style
  -------------
  - We can apply inline style using style attribute of a tag.

  - Inline style generally used when yu try to provide style to a particular tag.

  - Syntax :

       property : value

  - Example:

     <p style="color:red;background-color:yellow">This sample pragraph<p/>

     <p >This sample pragraph<p/>


     <p >This sample pragraph<p/>

- Internal Style
    
 - We can apply Internal style using style tag inside head tag

 - It is used when we need to apply style to more than one element at a time within same webpage

 - Syntax :

      <head>
         <style>
            selectors {

                property:value;
                property:value
            }
         </style>

      </head>

 - Example :
      <head>
         <style>
            h1{
                color:blue;
                background-color:yellow;
                border: 2px solid red;
                padding:10px;
            }
         </style>
      </head>

           }

       5. nth selector

3. External CSS 

    a. Create a css file with extension <filename>.css, whcih is called a stylesheet.

    b. Once you Create style sheet then you can place all css property using appropriate
       selectors.

       <link rel="stylesheet" type="text/css" href="style2.css" />

    Within a page you can link multiple external css

Selectors :

1. Core selector types

| Selector  | Example         | Targets                     |
| --------- | --------------- | --------------------------- |
| Type      | `p {}`          | all `<p>` elements          |
| Class     | `.btn {}`       | elements with `class="btn"` |
| ID        | `#hero {}`      | the element with `id="hero"`|
| Universal | `* {}`          | all elements                |
| Grouping  | `h1, h2, h3 {}` | multiple selectors at once  |

2. Combinators (relationships)

| Combinator       | Syntax      | Meaning                                 |
| -----------------| ----------  | ----------------------------------------|
| Descendant       | `.card p`   | any `<p>` *inside* `.card`              |
| Child            | `.card > p` | `<p>` that is a direct child of `.card` |
| Adjacent sibling | `h2 + p`    | first `<p>` immediately after an `<h2>` |
| General sibling  | `h2 ~ p`    | all `<p>` siblings after an `<h2>`      |

/* type selector */
h1 { color: #1a237e; }

/* class selector */
.member { background: #f3f3f3; padding: 1rem; }

/* id selector */
#site-header { background: #e8eaf6; }

/* grouping */
h2, h3 { color: #3949ab; }

/* descendant */
.team p { font-style: italic; }

/* child */
.team > .member { border-radius: 8px; }

/* adjacent sibling */
h3 + p { color: #616161; }

/* multiple class */
.member.featured { background: #c5cae9; }
